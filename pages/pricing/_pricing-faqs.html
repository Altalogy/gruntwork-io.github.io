Can I try Gruntwork Software before committing to an annual contract?

So, you just want a sneak peek? Before any contracts are signed, our team will give you a run-down of our code, training, and process. You can also try out parts of the IaC Library for free by checking out our open source modules.


What's a user?

A user is any human or machine user that makes use of Gruntwork's services, including accessing code from the IaC Library or Usage Patterns Library, watching videos in the DevOps Training Library, or submitting requests to Dedicated Support.

Is an annual subscription required?

The Gruntwork Subscription requires an annual contract (note: billing is monthly) because we give customers access to all the code in the IaC Library, and we let you keep that code even if you cancel. If we didn't have the annual contract requirement, customers could sign up for 1 day, get access to all the code, and cancel immediately. The Dedicated Support Plan doesn't have these issues, so it's a monthly contract with no annual commitment required.


What payment options do you accept?

For monthly billing, we require automated payments via credit card or ACH transfer. If you wish to use invoices and pay by wire transfer or check instead, you must do annual billing.


What happens if I cancel?

The Gruntwork Client License gives you perpetual rights to the code, so even if you cancel your subscription, you can keep using the code! You'll have one month to fork all the repos in the Infrastructure as Code Library into your own version control system, after which you'll no longer have access to the updates, additions, and fixes we make to the Infrastructure as Code Library, support from the Gruntwork team, security alerts, etc. However, your infrastructure will continue to work just fine and you'll still have all the code you were using up to that point, so there is no lock-in whatsoever.
