Reference Architecture

Most software teams share a common set of needs.

If your team is like most, you need some combination of a Docker cluster, CI/CD, database, cache, secrets management, monitoring, log aggregation plus a few other items. Once you tell us which tools we should use for each of these (e.g. Amazon ECS or Hashicorp Nomad? Jenkins or CircleCI?) we’ll write battle-tested, best-practices infrastructure-as-code that implements a Reference Architecture built specifically for your team.

How to Use It

Once you see the code for the Reference Architecture, it’s often straightforward to adapt these patterns to your unique needs, including:

  • Setup your microservices
  • Add additional AWS resources
  • Customize your CI builds
  • Tweak monitoring alerts
  • Build custom dashboards for your infrastructure

Available Options

With your reference architecture, choose from the following options:

Single AWS account or multiple AWS accounts

We support both single-account and multi-account setups.

Create any number of environments

Most customers need dev, stage, and prod, but add any environments you want.

Select your database

Select any RDS database (PostgreSQL, MySQL, MariaDB, Amazon Aurora, Oracle, SQL Server), or use our MongoDB Infrastructure Package.

Select your cache

Select any ElastiCache coaching engine (Redis or Memcached).

Select your Docker cluster

Choose Amazon ECS, HashiCorp Nomad, soon Kubernetes, or no Docker Cluster.

Bastion Host, Open VPN, or both?

Do you prefer to enter your network via a hardened Bastion Host, or via Open VPN? Either way, you can easily manage user access with Amazon IAM Groups.

Many more!

As part of our process, we’ll ask you to fill out a simple online form that presents a full range of options and takes about 5 minutes to complete.

What if I want something that’s not in your Reference Architecture?

If you want an option that we don’t yet support, if it’s a generic piece of infrastructure that other teams might also want, then we’ll build it for you. Additional fees may apply depending on your needs.

What does the Reference Architecure code look like?

All cose is written in open source, industry-standard tooling like Terraform, Bash, Go, or Python. The code is well-commented, well-documented, and written to best practices. By learning our code patterns, you’ll also learn best practices that will serve you in any position, whether Gruntwork is involved or not.

What’s included in the Reference Architecture?

Our deliverables to you will include:

  • A complete, customized reference architecture implemented in infrastructure-as-code
  • All code lives in your team’s git repo
  • The architecture is applied to your AWS account(s)
  • Comprehensive, personalized documentation
  • Live walkthrough with a Gruntwork engineer
