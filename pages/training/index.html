Training

Accelerate your DevOps learning curve.

We’ve spent years building up expertise in DevOps, system architecture, scalability, software delivery, security, Amazon Web Services, and more. With Gruntwork training, you can learn about all of these topics in a matter of hours or days.

Training we offer

We offer two types of training.

  • Crash courses
  • Workshops

Crash Courses

Our crash courses are the fastest way to get started with a new technology:

  • Each crash course lasts 1-2 hours.
  • The courses include instructor-led lectures and live-coding demonstrations.
  • Crash courses can be done remotely via a video conference or in-person at your office.
  • At the end of the course, you get access ot the slide deck, sample code, a list of resources where can get more info, and a recording of the training session.

We offer crash courses on the following topics:

Terraform

Get up and running quickly with Terraform by learning the syntax of HCL, how to manage state, how to package infrastructure into reusable Terraform modules, how to fit Terraform into your CI/CD workflow, and best practices for using Terraform as a team.

Packer, Docker, and ECS

Learn how to package your code into immutable images that run exactly the same way everwhere using Packer and Docker, as well as how to run Docker images in AWS with the EC2 Container Service (ECS).

AWS Fundamentals

This course walks you through all the basics of AWS, including how to use EC2, IAM, Auto Scaling Groups, Elastic Load Balancing, EC2 Container Service, RDS, S3, SQS, Lambda, and more.

DevOps Fundamentals

A practical guide to setting up a software deliver process ot take your code from dev to prod in a why that is automated, reliable, and secure. The course covers continuous integration, continuous delivery, automated tests, automated deployment, dependencies, microservices, infrastructure as code, feature toggles, monitoring, alerting, log aggregation, high availability, scalability, and more.

Security Fundamentals

Lean security best practices, including how to manage passwords for yourself and your team, expose secrets to your applications, manage user accounts and permissions with AWS in IAM, define a secure network topology in you VPCs, control access to servers with SSH and VPN, harden servers with permission settings and intrusion detection, and more.

Workshops

Workshops are longer, hands-on training sessions that give you a chance to go much deeper with each technology:

  • Each workshop lasts 1-2 days
  • All workshops are done in-person at your office. We also offer some of our workshops at O’Reilly’s Velocity conferences around the world.
  • The workshops include instructor-led lectures, live coding demonstrations, as well as coding exercises where you try out the technologies yourself on your own computer.
  • At the end of the workshop, you can spend 1-2 hours with the instructor doing a deep-dive on your company’s infrastructure to discuss how the new techniques you learned can be applied to your use cases.
  • After training, you get access to the slide decks, sample code, a list of resources where you can get more info, and a recording of the session.
  • You also get 5 hours of support from Gruntwork, which you can use to ask follow-up questions, request code reviews, and so on.

We offer workshops in the following topics:

Infrastructure as Code with Terraform (2 days)

Get up and running quickly with Terraform using real-world examples. You’ll deploy servers, databases, and load balancers on AWS; you’ll build immutable infrastructure with Docker and Packer, and you’ll put it all together into continuous delivery pipeline. This workshop covers Terraform’s syntax, the CLI, state management, modules, team workflows, environment, management, wesioning, gotchas, and more.

The modern end-to-end tech stack (2 days)

A whirlwind tour of all the pieces you need to create a modern tech stack. Learn about server clusters and Docker clusters; server-side frameworks and client-side frameworks; relational databases, NoSQL, databases, schema management, and stream processing; monitoring, alerting, and log aggregation; network security, secret storage, TLS, SSH, and VPN; continusous integration, continuous delivery, automated deployment, feature toggles, and dark launches; static content, CDNs, and versioning; scalability, high availability, and performance, and more.
