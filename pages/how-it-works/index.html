How it Works

  Watch the Overview Video

1. Tell us your infrastructure needs with our free online form.

Fill out our simple, free online form to tell us what infrastructure from the Gruntwork Library and Reference Architecture you want to use: Docker or Packer? MySQL or PostgreSQL? Jenkins or CircleCI? CloudWatch or DataDog? KMS or Vault? Bastion host or VPN server?

2. Schedule a free live walkthrough.

We’ll create the infrastructure you requested in Terraform, Go, Bash, and Python using code from the Gruntwork Library and our Reference Architecture, and set up a free video call with you to walk you through the code you’d be getting. You’ll be able to see exactly how the code looks, how it’s documented, how it’s tested, the different ways you can configure, compose, and reuse it, and how it works when running in an AWS account.

3. If you like what you see, become a Gruntwork Subscriber.

If the code looks like a good fit, we’ll sign you up as a Gruntwork subscriber, get your payment details, transfer the code into your Git repos, and deploy it into your AWS account, getting you up and running with a battle-tested, end-to-end infrastructure—all in about a day! If the code doesn’t fit your needs, you don’t pay a cent.

4. Learn how to use your new infrastructure.

The infrastructure code comes with written and video documentation of how to do all the common DevOps tasks: how to run the code in dev, how to build and test, how to deploy changes, where to find metrics, logs, and alerts, how to SSH to servers, and so on. If you want to go even deeper, sign up for one of the Gruntwork Training Course on Terraform, AWS, Docker, Packer, Security, and so on.

5. Keep your infrastructure up to date.

As a Gruntwork Subscriber, you get access to all the updates and additions we make to the Gruntwork Library, so better infrastructure is just a version number bump away. Moreover, if you sign up for Gruntwork Patcher(beta), we will automatically update your code and submit pull requests to ensure you stay up to date with the latest changes from the Gruntwork Library, as well as AWS, Terraform, Docker, and other critical infrastructure technologies.
